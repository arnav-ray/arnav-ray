Pull vs Push: The Two Architectures of Agentic AI

How enterprises access AI capability -- and why the split matters for SAP customers

Architecture A Pull
What it is
Agent-initiated, tool-mediated access
The AI agent reaches into enterprise systems at runtime, using standardised tool APIs to retrieve data and execute actions on demand.
Key protocol
Model Context Protocol (MCP)
Anthropic's open standard (2024), now widely adopted. Defines how agents discover and call tools across system boundaries without custom integration work per pair.

Clean Core dependency
Hard requirement via SAP API Policy v4
Only stable, released SAP APIs are permitted as MCP tool endpoints. Custom ABAP that modifies standard objects is invisible -- or worse, unstable -- to external agents.
Opportunity
Open agent ecosystem
Any agent -- internal or third-party -- can be pointed at your SAP system if you expose clean APIs. Clean Core becomes a procurement signal, not just a technical posture.
Architecture B Push
What it is
Vendor-embedded, workflow-triggered AI
The software vendor embeds AI directly into existing workflows. The system pushes AI suggestions into the user's context at defined moments -- no separate agent required.
SAP's implementation
Joule + SAP Business AI
Joule is SAP's embedded copilot across S/4HANA, Ariba, SuccessFactors and others. Skills are pre-built workflow integrations. Joule Studio lets customers configure and extend them.

Clean Core dependency
Softer, but real
Joule skills target standard SAP processes. Heavy customisation can break skill applicability or require manual re-mapping after every upgrade -- the same migration tax as before, now affecting AI features.
Constraint
SAP controls the roadmap
Push AI ships on SAP's release cadence. Customers influence -- but do not direct -- which skills appear and when. The upside is zero integration cost; the tradeoff is reduced optionality.
What Clean Core connects
API stability
SAP API Policy v4 restricts MCP tool endpoints to stable, released APIs. Clean Core is the prerequisite.
Agent portability
A clean system can be reached by any MCP-compatible agent. A customised system is reachable only by agents written for its quirks.
Upgrade leverage
Clean Core preserves the ability to take SAP innovations -- Joule skills, new APIs -- without costly remediation on every release.
